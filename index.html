How does Javascript works?

Javascript is a single threaded programming language. Javascript handle multiple tasks asynchronously. When we run javascript code, it will create an event loop. Javascript create a stack of multiple tasks. When there comes an asynchronous tasks, js will make a queue of asynchronous tasks. Then all the tasks are executed from the stack and the queue in order.

What does Javascript event loop do?

To do asynchronous task js will use event loop. In event loop there are stack for synchoronous/sequential tasks, heap where data stored randomly and memory allocated, and queue(waiting list of asynchoronous tasks). Javascript handle multiple tasks using event loop. An event loop is something that pulls tasks out of the queue and places it onto the function execution stack whenever the function stack becomes empty. JavaScript has a model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.

What are the differences between Local Storage and Session Storage?

Session storage and Local storage are the concepts of Web API. These two are similar. The main difference between these two are, when we close the browser or browser tab session storage data got expire or erased from the browser. When browser page session ends, session storage cleared. But local storage data do not erased or expired from the browser.
